Are good developers more focused? Data-mining GitHub.

Submission List

Folder structure:
  1. Articles - related works
    • DejaVu
    • Dual Ecolofical Measures of Focus
    • When do Changes Induce Fixes
    • The Promises and Perils of Mining GitHub
    • The Sky is Not the Limit: Multitasking Across GitHub Projects
    • StackOverflow and GitHub: Associations Between Software Development and Crowdsourced Knowledge
  2. Documentation folder
    • Spec Document - Technical Specs
    • Repositories Criteria - the repo characteristics considered when selecting repositories for the project
    • Selected Repositories
    • Repositories considered for selection - list of the repositories considered for selection with their characteristics
  3. Tools folder:
    • instructions.txt - a file that describes the prerequisites and the order in which scripts should be run
    • python scripts for extracting and analyzing data. They are all described in the Spec Document, and they contain in-file documentation. To see the order they need to be run in, check instructions.txt
    • tests - contains test scripts
    • data folder - contains research data
      • json files - combined data for all the repositories
      • test folder - folder for test data for scripts that are run on the combined data from all the repositories
      • ansible data - research data for ansible
      • bitcoin data - research data for bitcoin
      • elasticsearch data - research data for elasticsearch
      • rails data - research data for rails
      • rust data - research data for rust
      • selenium data - research data for selenium
      • servo data - research data for servo
      • spring-boot data - research data for spring-boot
      • symfony data - research data for symfony
      • typescript data - research data for typescript
    • repos folder - empty folder where the repositories should be cloned (check instructions.txt)
    • Visualization folder
      • developers-focus.html - page to visualize the focus analysis results
      • developers.html - page to visualize the inserted (good and buggy) lines of every devleoper in a selected repository
      • developers_ratios.html - page to visualize the ratios of every devleoper
      • charts folder - contains pdf of the focus charts
        • static focus charts
      • data folder - contains data to be visualized
      • scripts folder - contains js scripts for visualization
What is not included in the submission:
  1. Cloned repositories (see instructions.txt)
  2. Test data under data/<repo>/test/ and data/test/ (Run all the scripts with test flag set to True according to the instructions.txt to get the test data)
  3. Issues data under data/<repo>/issues.json. To get issues data run download_issues.py for each repo according to the instructions.txt
